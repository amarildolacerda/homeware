# Design: ESP8266 PIR Presence Detector

## Visão Geral
Cliente ESP8266 puro sensor — detecta presença via HC-SR501, envia estado (PRESENTE/AUSENTE) ao gateway via ESP-NOW. Sem atuação (sem relé/LED externo).

## Hardware
| Componente | GPIO | Notas |
|------------|------|-------|
| PIR HC-SR501 | D2/GPIO4 | Saída digital HIGH = presença |
| LED built-in | D4/GPIO2 | Active LOW |

## Protocolo
- **Sensor type**: `SENSOR_TYPE_MOTION` (3) — já definido no gateway
- **Payload**: `payload_motion_t` — `motion_state` (uint8) + `occupancy_duration` (uint8)
- **Estado**: 0 = ausente, 1 = presença
- **Hold time**: configurável (padrão 30s), mantém estado PRESENTE após última detecção

## Dashboard
- **Hero**: Badge grande "PRESENTE" (verde) / "AUSENTE" (vermelho)
- **Stats Header**: RX, TX, Mem, +1 stat (uptime)
- **Detalhes collapsível**: Gateway, RSSI, IP, Uptime, Versão
- **Footer**: dot gateway, horário, uptime
- Segue layout compacto padrão (sidebar + collapsible)

## config.h
```cpp
#define FW_VERSION "v0.0.22"
#define DEVICE_NAME "PIR ESP-NOW"
#define PIR_PIN 4              // GPIO4/D2
#define STATE_UPDATE_INTERVAL 5000
#define PIR_HOLD_TIME_MS 30000 // 30s hold time
#define HEARTBEAT_INTERVAL 60000
#define LED_PIN 2
```

## main.cpp — Non-blocking Loop
```
loop():
  - Lê PIR a cada 1s (debounce)
  - Se HIGH → s_last_motion = millis()
  - Estado = (millis() - s_last_motion < PIR_HOLD_TIME_MS)
  - Envia ao gateway se estado mudou ou a cada STATE_UPDATE_INTERVAL
  - Heartbeat a cada 60s
  - LED: blink rápido (WiFi), blink lento (não pareado), OFF (pareado)
```
